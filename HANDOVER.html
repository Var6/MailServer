1. Project Overview

MailServer is a fully self-hosted productivity suite — a drop-in alternative to Gmail / G Suite that runs entirely on your own hardware. It provides webmail, shared calendar, contacts, file storage, and in-browser office document editing.

📧 Webmail

Gmail-style 3-pane inbox. Compose, reply, folders, search. Powered by Dovecot (IMAP) + Postfix (SMTP).

📅 Calendar

CalDAV sync. Shared team events. FullCalendar UI with drag-and-drop.

👥 Contacts

CardDAV-based contact management. Syncs with phone / Outlook / Thunderbird.

📁 Files

Upload, browse, share files & folders (Google Drive style). Stored in D:\maildata\files\.

📝 Office Editing

Edit .docx / .xlsx / .pptx in-browser via Collabora CODE (WOPI protocol). No desktop app needed.

🔒 Multi-tenant

Superadmin creates company tenants. Each company admin manages their own users.

Tailwind & Dynamic Colors

⚠️ Tailwind JIT purges unused classes at build time. If you construct class names dynamically (e.g. `bg-${color}-500`), Tailwind won't include them. Always use inline styles for dynamic colors: style={{ backgroundColor: color }} where color is a hex string.

Adding a New API Call

  2. Use with useQuery (read) or useMutation (write) from TanStack Query
  3. The axios client in api/client.ts handles JWT tokens and auto-refresh automatically

9. Backend Guide

The backend is a Node.js Express REST API written in TypeScript, compiled with tsx in development and tsc for production Docker builds.

PrefixFileWhat it does

GET /files/listroutes/files.tsBrowse user files
POST /files/uploadroutes/files.tsUpload a file
POST /files/shareroutes/files.tsShare file/folder with another user
GET /files/shared-with-meroutes/files.tsList files shared to current user
GET /calendar/eventsroutes/calendar.tsFetch CalDAV events
GET /contactsroutes/contacts.tsFetch CardDAV contacts

GET /admin/usersroutes/adminPanel.tsList users (admin only)
GET /superadmin/tenantsroutes/tenants.tsList tenants (superadmin only)

Authentication Flow

  1. Client sends POST /auth/login with email + password
  2. Backend verifies against Dovecot (IMAP login attempt) or MongoDB for admin/superadmin
  3. Returns short-lived access token (15 min) in response + long-lived refresh token in httpOnly cookie
  4. Client stores access token in Zustand (memory only — not localStorage)
  5. On 401, the axios interceptor in api/client.ts auto-calls POST /auth/refresh

Adding a New API Route

  1. Create backend/src/routes/myRoute.ts
  2. Import and register in backend/src/index.ts: app.use('/my-route', myRouter)
  3. Protect with JWT middleware: router.use(requireAuth) from middleware/auth.ts
  4. Add Zod schema validation for request body

10. Database Guide

MongoDB

Used for: User accounts, Tenants, File shares, Billing. Mail data (actual emails) is stored by Dovecot in D:\maildata\mail\ in Maildir format.

# Connect to MongoDB in Docker
docker exec -it mailserver-mongodb mongosh \
  -u mailserver -p changeme \
  --authenticationDatabase mailserver

# Common queries
use mailserver
db.users.find()           # List all admin/superadmin accounts
db.tenants.find()         # List all company tenants
db.shares.find()          # List all file shares
⚠️ In development with bash dev.sh, the backend connects to a LOCAL MongoDB instance (no auth) at localhost:27017. In Docker, it connects to mongodb:27017 with auth credentials from .env.

Redis

Used for: JWT refresh token allowlist / blacklist (session invalidation on logout).

# Connect to Redis in Docker
docker exec -it mailserver-redis redis-cli -a $REDIS_PASSWORD

14. File Storage

Files are stored on the Windows host at D:\maildata\files\ and mounted into the api Docker container at /data/files.

Directory Structure

D:\maildata\files\
  ├── user-at-domain-com\      ← sanitized email (@ → -at-, . → -)
  │   ├── documents\
  │   ├── Sent\               ← attachments from sent emails
  │   └── received\
  │       └── sender-at-domain-com\  ← attachments saved from received emails
  └── anotheruserat-domain-com\

File Sharing

Sharing is implemented in backend/src/routes/files.ts using the Share Mongoose model. The share record stores: ownerEmail, path, sharedWith (array of emails), permission (view/edit).

Shared files appear in the "Shared with me" tab in the Files page.

15. Office Editing (WOPI / Collabora)

Clicking "Edit in Office" on a .docx, .xlsx, or .pptx file opens Collabora CODE editor in the browser. This uses the WOPI protocol.

Flow

1. Frontend calls POST /wopi/token  →  gets {token, wopiSrc, editorUrl}
2. Frontend opens Collabora at {editorUrl}?WOPISrc={wopiSrc}&access_token={token}
3. Collabora fetches file from  GET  /wopi/files/{fileId}     (our API)
4. Collabora saves file back to PUT  /wopi/files/{fileId}     (our API)
5. Our API reads/writes from D:\maildata\files\{user}\{path}

Config Variables

VariableDocker valueDev value
COLLABORA_URLhttp://collabora:9980http://localhost:8082
WOPI_HOSThttp://api:3000http://host.docker.internal:3000
In dev mode, Collabora (running in Docker) needs to reach the backend on the host machine. It uses host.docker.internal (Windows Docker Desktop resolves this to the host IP automatically).

16. Common Errors & Fixes

ErrorCauseFix
Changes not visible after editing code Accessing port 80 (Docker container) instead of port 5173 (dev server) Use http://localhost:5173 in dev. OR rebuild Docker container and hard refresh (Ctrl+Shift+R)
Cannot find module 'tsx' Corrupt or missing node_modules in backend cd backend && rm -rf node_modules package-lock.json && npm install
Port 5173 only accessible on IPv6 Vite not binding to IPv4 Ensure vite.config.ts has server: { host: "0.0.0.0" }
"Could not open editor" in Collabora WOPI route not registered, wrong host config, or .env not loaded Check COLLABORA_URL and WOPI_HOST env vars. Ensure backend/.env exists and has correct values. Restart backend.
MongoDB auth failing in dev Local MongoDB has no users; Docker MongoDB requires credentials Dev backend/.env should use mongodb://localhost:27017/mailserver (no auth)
Avatar/profile not loading Token refresh wiping avatar from store Fixed: api/client.ts refresh call preserves cur.avatar
Avatar color (Tailwind class missing) Tailwind JIT purges dynamic class names Use style={{ backgroundColor: hexColor }} instead of dynamic Tailwind classes
Rollup missing (Windows install) Corrupt package-lock with wrong binary cd frontend && rm -rf node_modules package-lock.json && npm install
Docker build fails on Windows paths Windows backslash in docker-compose volume path Use forward slashes: D:/maildata/files:/data/files
